1 Создание описания с помощью графического редактора

 Специализированный графический редактор позволяет создать подробную модель ДКА.

 Нажатие ПКМ на пустом месте этого редактора открывает контекстное меню, с помощью которого можно добавить вершину графа (Рисунок 1). При нажатии ПКМ по созданному состоянию появляется контекстное меню состояния, с помощью которого, выбрав нужный пункт ЛКМ, можно добавить стрелку графи и настроить само состояние (Рисунок 2). Аналогично можно выбрать и сконфигурировать параметры перехода (Рисунок 3):

 

Рисунок 1 – Нажатие ПКМ по пустому пространству

 

Рисунок 2 – Нажатие ПКМ по состоянию

 

Рисунок 3 – Нажатие ПКМ по переходу

 При выборе пункта конфигурации “Установить имя” состояния или “Установить сигнал” перехода появляется диалог (Рисунок 4), который по нажатию Enter отправляется заданный вход на обработку программе (Рисунок 5).

2 Выбор представления описания

 Наведя на пункт “Построить” в меню главного окна и нажав ЛКМ появятся варианты представления ДКА (Рисунок 6).

 

Рисунок 6 – Выбор варианта представления

 Выбрав пункт “Таблица” с помощью ЛКМ, откроется новое окно с построенной таблицей переходов ДКА. Аналогично откроется окно с построенным кодом по нажатию на “Код” (Рисунок 7). Оба эти представления позволяют манипулировать построенной моделью ДКА стандартными средствами.

3 Анализ ДКА

 Анализировать ДКА с помощью программы можно двумя способами:

  проверить входную цепочку строк на корректность;

 минимизировать таблицу перехода.

 Первый вариант доступен через пункт меню “Проверить входные данные”. При нажатии на него появляется форма ввода с подсказкой формы ввода для пользователя (Рисунок 8). После принятия ввода программа выведет информацию о корректности входной последовательности (Рисунок 9).

4 Сохранение представлений в файл

 Каждое 3-х рассмотренных представлений предоставляет возможность сохранить себя в соответствующем формате.

 Нажав на пункт меню главного окна “Файл” следует выбрать “Сохранить как” (Рисунок 10). Тогда появится файловый диалог, с помощью которого можно сохранить граф состояний в формате svg.

 

Рисунок 10 – Сохранение графа

 В табличном представлении следует нажать с помощью ЛКМ кнопку “Сохранить”, которая также открывает файловый диалог, с помощью которого можно сохранить таблицу переходов в формате csv (Рисунок 11).

 

Рисунок 11 – Файловый диалог сохранения таблицы

 В кодовом представлении следует нажать ПКМ и затем с помощью ЛКМ выбрать пункт “Сохранить код” из появившегося контекстного меню (Рисунок 12), что позволит сохранить код ДКА в формате cpp.
